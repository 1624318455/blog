# 推送代码指南

## 🚀 快速推送（推荐）

```bash
.\push.ps1
```

**流程**：
1. ✅ 自动运行 `npm run validate`
2. ✅ 验证通过 → 自动推送到 GitHub
3. ✅ 验证失败 → 显示错误，停止推送

---

## 📋 完整推送（详细日志）

```bash
.\push-with-check.ps1
```

显示更详细的推送过程和每个文件的状态。

---

## 🔧 手动验证（不推送）

```bash
npm run validate
```

只检查代码质量，不推送。

---

## ⚡ 工作流程

### 开发阶段
```bash
npm run dev          # 本地开发
npm run lint:fix     # 自动修复问题
```

### 提交前
```bash
npm run validate     # 完整验证
```

### 推送代码
```bash
.\push.ps1          # 一键验证 + 推送
```

### 部署
```
Vercel 自动部署（通常 1-2 分钟）
```

---

## 📊 脚本说明

| 脚本 | 功能 | 用途 |
|------|------|------|
| `push.ps1` | 快速推送 | 日常使用，简洁输出 |
| `push-with-check.ps1` | 完整推送 | 详细日志，调试用 |

---

## ✅ 检查项

推送前会自动检查：

- ✅ TypeScript 类型错误
- ✅ 未使用的变量
- ✅ ESLint 代码风格
- ✅ 导入/导出问题

---

## 🎯 从今以后的推送流程

```
修改代码
    ↓
.\push.ps1 (一键推送)
    ↓
自动验证 (npm run validate)
    ↓
验证失败? → 显示错误，停止
    ↓
验证通过? → 自动推送到 GitHub
    ↓
Vercel 自动部署 (通常成功)
```

---

## 💡 优势

1. **完全避免部署失败** — 代码质量有保障
2. **节省时间** — 不用等 Vercel 失败再修复
3. **自动化流程** — 一键完成验证 + 推送
4. **清晰的反馈** — 实时显示检查结果

---

## 🚨 如果推送失败

1. 查看错误信息
2. 运行 `npm run lint:fix` 自动修复
3. 手动修复无法自动修复的问题
4. 再次运行 `.\push.ps1`
